# Support

Need help with ACDD? Here are several ways to get support:

## Documentation

Start with our documentation:
- [Quick Start Guide](./docs/quickstart.md) - Get up and running quickly
- [Installation Guide](./docs/installation.md) - Complete installation instructions
- [Local Development](./docs/local-development.md) - Development setup and testing

## Issue Tracker

For bugs, feature requests, or questions:
- Check the [existing issues](https://github.com/acdd/acdd-kit/issues) first
- If you don't find your issue, [create a new one](https://github.com/acdd/acdd-kit/issues/new)
- Be descriptive and include steps to reproduce if reporting a bug

## Community Support

- GitHub Discussions (when available) - General questions and community help
- The issue tracker is also used for questions when discussions aren't available

## For Maintainers

If you're interested in contributing or becoming a maintainer:
- See [CONTRIBUTING.md](CONTRIBUTING.md) for contribution guidelines
- Open an issue to introduce yourself and express interest

## Troubleshooting Common Issues

### Installation Problems
- Ensure Python 3.11+ is installed
- Check that you have the necessary permissions
- Verify that pip/uv is up to date

### Script Execution
- Make sure generated scripts have execute permissions
- Check that required tools (git, etc.) are in your PATH
- Verify that AI assistant integration is set up correctly

## When to Reach Out

- You've found a bug in ACDD
- You have a feature request or enhancement idea
- You're having trouble getting started
- You need clarification on ACDD methodology
- You want to contribute to the project

We're here to help make your ACDD experience successful!
