# ACDD (Adaptive Context-Driven Development) Template Summary

## Overview

This document provides a comprehensive overview of the ACDD template implementation that extends the existing spec-kit methodology with adaptive capabilities. The ACDD approach enables software systems to evolve and adapt based on changing contexts while maintaining architectural integrity and pattern compliance.

## Key Components

### 1. ACDD Tasks Template

- **File**: `acdd-tasks-template.md`
- **Purpose**: Provides the structure for generating adaptive task lists with context tracking
- **Features**:
  - Context-aware task organization
  - Adaptive implementation phases
  - Pattern compliance validation
  - Context evolution tracking

### 2. ACDD Tasks Command Template

- **File**: `commands/acdd-tasks.md`
- **Purpose**: Defines the command structure for the `/acdd.tasks` command
- **Features**:
  - Context pattern analysis
  - Learning preparation tasks
  - Adaptive foundation setup
  - Quality validation points

### 3. Validation Scripts

- **Bash**: `scripts/bash/acdd-validate.sh`
- **PowerShell**: `scripts/powershell/acdd-validate.ps1`
- **Purpose**: Validates ACDD-specific requirements and context tracking
- **Features**:
  - Context map validation
  - Pattern compliance checking
  - Context tag verification
  - JSON output support

### 4. Quality Checklist

- **File**: `acdd-checklist.md`
- **Purpose**: Provides a comprehensive checklist for ACDD implementation quality
- **Features**:
  - Phase-by-phase validation
  - Context evolution tracking verification
  - Pattern compliance validation
  - Adaptive implementation verification

## ACDD Methodology Phases

### Phase 1: Context Setup & Pattern Recognition [CTX-INIT]

- Analyze existing context patterns
- Set up project structure with context tracking
- Configure context evolution tools

### Phase 2: Learning Preparation [CTX-LEARN]

- Research context patterns from previous features
- Identify context change triggers
- Prepare validation criteria and monitoring

### Phase 3: Adaptive Foundation [CTX-ADAPT]

- Set up adaptive infrastructure
- Implement context-aware authentication/authorization
- Create adaptive base models with context tracking

### Phase 4+: User Stories [CTX-ADAPT/CTX-VALIDATE]

- Implement adaptive user stories
- Create context-aware tests
- Validate pattern compliance

### Final Phase: Adaptive Polish & Context Validation [CTX-VALIDATE]

- Update documentation with context awareness
- Perform adaptive refactoring
- Validate context evolution tracking

## Context Tags

The ACDD methodology uses specific context tags to track the adaptive development process:

- **[CTX-INIT]**: Initial context setup and pattern recognition
- **[CTX-LEARN]**: Learning preparation and knowledge acquisition
- **[CTX-ADAPT]**: Context adaptation and implementation adjustments
- **[CTX-VALIDATE]**: Validation against evolving context requirements
